Contenido tomado de []

  • Listas (list): estructuras flexibles que permiten agrupar objetos de cualquier tipo, incluso mezclados. Son mutables, lo que significa que pueden modificarse despu√©s de ser creadas. Se usan com√∫nmente para almacenar colecciones ordenadas de elementos.
    üìå Ejemplo: ["manzana", 3, True]

  • Tuplas (tuple): estructuras similares a las listas, pero inmutables (no se pueden modificar una vez creadas). Se utilizan cuando se quiere agrupar datos de diferentes tipos que no deben cambiar, como coordenadas o registros fijos.
    üìå Ejemplo: (37.5, "Latitud", True)

  • Arrays (numpy.array): estructuras optimizadas para almacenar y operar sobre datos num√©ricos homog√©neos. Son m√°s eficientes que las listas para c√°lculos matem√°ticos y cient√≠ficos. Aunque existen objetos como matrix, la mayor√≠a de funciones de NumPy y bibliotecas cient√≠ficas prefieren los arrays.
    üìå Ejemplo: np.array([1.5, 2.3, 3.1])

  • Diccionarios (dict): estructuras que almacenan pares clave‚Äìvalor. Son ideales para representar datos estructurados o asociaciones, como atributos de una persona o propiedades de un objeto.
    üìå Ejemplo: {"nombre": "Ana", "edad": 30}

  • DataFrames (pandas.DataFrame): estructuras tabulares que combinan la funcionalidad de arrays y diccionarios. Son especialmente √∫tiles para an√°lisis estad√≠stico y cient√≠fico, ya que permiten manipular, filtrar y resumir grandes vol√∫menes de datos con etiquetas.
    üìå Ejemplo: tabla con columnas nombre, edad, y ingreso.

1.4. Indexaci√≥n & Slicing#

Las reglas para acceder elementos individuales en listas, tuplas o arrays de numpy son:

 #a[ini:fin] # items desde ini hasta fin-1
 # a[ini:]    # items desde ini hasta el resto del array
 # a[:fin]    # items desde el inicio hasta el final -1
 # a[:]       # todo el array`

Se puede usar tambi√©n step con cualquiera de las anteriores:

a[ini:fin:step] #empieza en ini hasta fin-1 separados por step

El punto clave a recordar es que la indexaci√≥n empieza en 0, NO en 1; y que :fin representa el primer valor que NO es seleccionado. Los √≠ndices tambi√©n pueden ser negativos (ini o fin) que indica que se cuenta desde el final del array y no desde el principio

#a[-1] #ultimo √≠tem del array
#a[-2:] #dos ultimos √≠tems del array
#a[:-2] #todo excepto los dos √∫ltimos √≠tems`

Tambi√©n a[:5] devuelve los 5 primeros elementos, y a[-5:] los √∫ltimos.

Practica la indexaci√≥n con el siguiente array:

a = [5,4,3,6,8,62,1,2,3,6,8,9]

a[-1:]

[9]

a[-2:]

[8, 9]

a[:-2]

[5, 4, 3, 6, 8, 62, 1, 2, 3, 6]

a[2:12:2]

[3, 8, 1, 3, 8]

len(a)

12

### CONTINUE

Esta sentencia, continue, permite saltarse una porci√≥n de c√≥digo en un loop iterativo. Si el int√©rprete encuentra un continue regresa al principio del loop sin ejecutar las l√≠neas debajo de continue.

El siguiente ejemplo compila una lista de todos los n√∫meros entre el 1 y el 99 que son divisibles para 7.

1.8.1. Funciones, m√≥dulos y paquetes#

  • Funci√≥n Una funci√≥n se define por la palabra clave def. Devuelve un objeto con la sentencia return que generalmente va al final.

  • M√≥dulos Es un archivo con la extensi√≥n .py. Los m√≥dulos pueden contener funciones y definiciones de variables.

  • Paquetes: Un paquete es una carpeta que contiene varios m√≥dulos y debe contener un archivo llamado __init__.py. Por ejemplo numpy es un paquete de python.

Funciones

'''Demostraci√≥n de una funci√≥n de python
author: thomas haslwanter, date: May-2015
'''

# Importamos los paquetes
import numpy as np

def incomeAndExpenses(data):
    '''Encuentre la suma de valores positivos, y la suma de valores negativos.'''
    income = np.sum(data[data>0])
    expenses = np.sum(data[data<0])
    return(income,expenses)

testData = np.array([-5,12, 3, -6, -4, 8])
(myIncome,myExpenses) = incomeAndExpenses(testData)

El n√∫mero de par√°metros en la definici√≥n de una funci√≥n puede ser arbitrario. Por ejemplo, en la definici√≥n

def func(x1,x2,*x3)

x1 y x2 son par√°metros usuales, tambi√©n llamados par√°metros posicionales, pero x3 es una tupla de longitud arbitraria que contiene par√°metros adicionales. Si llamamos a la funci√≥n

func(a,b,c,d,e)

entonces a corresponde a x1, b a x2 pero c,d,e corresponde a x3.
